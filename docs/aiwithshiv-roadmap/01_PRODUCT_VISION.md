# Product Vision

## Purpose

Define what AIWithShiv should become, who it serves, what it promises, and what should not be built yet.

## Mission

AIWithShiv helps Indian children become safe, curious, confident AI learners through playful missions with Shiv, a friendly robot-superhero tutor.

## Product Positioning

AIWithShiv is a local-first AI learning app for Indian children aged 5-10. It teaches AI basics through comic stories, quizzes, games, voice narration, XP, coins, trophies, and bilingual English/Hindi support.

## Audience

| Segment | Primary promise |
| --- | --- |
| Children | Learn AI like a superhero mission |
| Parents | Safe, private, offline-first learning |
| Teachers | Simple classroom-friendly AI concepts |
| Schools | Low-friction early AI literacy |
| EdTech partners | Structured local-first AI curriculum |

## Child Promise

"You can become an AI Hero with Shiv by learning, playing, checking answers, and staying safe."

## Parent Promise

"Your child can learn future-ready AI basics without login, phone number, email, password, or unnecessary internet access."

## School Promise

"AIWithShiv introduces AI literacy with child-safe content, short activities, local-first progress, and Indian examples."

## Core Value Proposition

| Value | How app delivers it |
| --- | --- |
| Safe AI learning | Privacy lessons, local progress, no child login |
| Fun learning | Comic UI, Shiv, games, rewards |
| Indian context | Hindi words, tiffin, cricket, Diwali, school examples |
| Beginner-friendly | Short stories and two-question checkpoints |
| Offline-first | JSON assets and SharedPreferences |
| Parent trust | No Firebase required for core learning |

## Unique Selling Points

- AI literacy for young Indian children.
- Friendly robot-superhero character.
- Local-first learning without child accounts.
- Bilingual English/Hindi direction.
- Comic mission structure instead of textbook theory.
- Rewards and games tied to AI concepts.

## Monetization Direction

| Option | Fit | Notes |
| --- | --- | --- |
| One-time paid app | Strong | Best for parent trust and child safety |
| Freemium with paid packs | Strong | Keep core free; charge for advanced courses |
| School licensing | Strong future option | Needs teacher dashboard/content pack |
| Optional child-directed ads | Risky | Already feature-flagged; use carefully |
| Subscription | Possible later | Needs more content depth |

## What Not To Build Yet

- Child login/signup.
- Social features or public leaderboards.
- Parent surveillance dashboard with sensitive child data.
- Direct AI API calls from Flutter in production.
- Open-ended child data collection.
- Ads inside lessons, quizzes, games, ShivBot, profile, or trophies.
- Advanced coding IDE features before the age 5-10 core is polished.

## Current Project Understanding

The current app already has the correct foundation: dashboard-first flow, local JSON, SharedPreferences, games, achievements, shop, TTS, localization, and optional ads. The product still needs stronger parent/teacher packaging and production safety hardening.

## What Already Exists

- Dashboard-first child experience.
- Three Phase 1 courses.
- Five games.
- XP, coins, streaks, badges, shop.
- English/Hindi framework.
- Optional ads feature flag.

## What Is Missing

- Store-ready product positioning.
- Parent/teacher trust content.
- School pilot materials.
- Privacy policy final copy.
- Clear monetization decision.

## Recommended Next Steps

1. Position Phase 1 as "AI basics for kids age 5-10."
2. Keep the first launch small and polished.
3. Build parent trust before monetization.
4. Treat ads as optional, disabled by default for launch unless compliance is complete.

- App positioning is understandable in one sentence.
- Parent trust message is clear.
- Product scope avoids account, privacy, and AI API risks.
- Launch plan supports Android and iOS.
